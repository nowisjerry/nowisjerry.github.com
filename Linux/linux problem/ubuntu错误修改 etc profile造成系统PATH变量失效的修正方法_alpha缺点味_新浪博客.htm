ubuntu错误修改/etc/profile造成系统PATH变量失效的修正方法

(2013-03-11 11:01:47)
转载▼
标签：

path变量

ubuntu环境变量

shell命令失效

 分类： Ubuntu
楼主昨日新装软件，但由于刚上手ubuntu系统不久，在修改/etc/profile增加PATH变量时，直接在文本最后添加
export PATH=/usr/lib/kde4/bin，结果造成系统原始PATH变量失效，如ls,sudo等基本命令都无法使用。

先给出解决方法，理论的东西有兴趣的看后面：
1.注意最好不要重新启动系统，否则只能通过shell登录，图形界面是无法登录的；
2.shell环境下输入命令 $/usr/bin/sudo /usr/bin/vim /etc/profile，进入到profile文件的编辑模式，删除之前的修改，在文本后面加入export PATH=/usr/sbin:/bin:/usr/bin:/sbin$PATH，退出并保存文件。
3.在shell输入$source /etc/profile或者重启一次系统，ok了，命令都回来了。

下面来说说原理:
1.在修改PATH变量时，如果在命令最后没有$PATH，系统就会清除之前的PATH路径变量，只保留最后输入的值；
2.正确的修改方法是在输入新的PATH变量时，通过:（冒号）来隔离两个变量；
3.Linux系统的shell命令是通过PATH变量来查找的，如果系统默认的PATH变量被删除，自然无法执行命令。

昨天博主为了这个问题，奋战了今两个小时，网上几乎没有解决方法（难道是没人遇到过？），博主只找到一篇，但是解决方法有点问题，故才有此篇。首次发博，不对之处，望斧正。
